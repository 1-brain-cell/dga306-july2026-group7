# ข้อจำกัดของโปรเจกต์

เอกสารนี้อธิบายข้อจำกัดสำคัญที่ควรอ่านร่วมกับผลการวิเคราะห์ เพื่อให้ตีความผลได้อย่างถูกต้อง

[กลับหน้าแรก](../)

---

## 1. ข้อมูลมีเพียง 77 แถว

ข้อมูลหลักที่ใช้วิเคราะห์คือข้อมูล LABAI ระดับจังหวัด ซึ่งมี **77 แถว** (หนึ่งแถวต่อจังหวัด) ข้อมูลขนาดนี้เพียงพอสำหรับการสำรวจ pattern เบื้องต้น แต่ไม่เหมาะสำหรับการฝึก machine learning ที่ต้องการความน่าเชื่อถือสูง และผลลัพธ์อาจไวต่อแถวที่มีค่าสูงผิดปกติบางแถว

---

## 2. ไม่มีข้อมูลช่วงเวลา จึงไม่สามารถทำ forecasting ได้

ข้อมูล LABAI ที่ใช้เป็น **snapshot ณ เวลาหนึ่ง** ไม่มี column วันที่หรือรอบการรวบรวมข้อมูล ดังนั้นโปรเจกต์นี้จึงไม่ทำการพยากรณ์แนวโน้ม (forecasting) เพราะข้อมูลชุดเดียวไม่เพียงพอสำหรับการสร้าง time series

---

## 3. ค่า `-` ในข้อมูลการจับคู่ยังไม่มีความหมายที่ยืนยัน

ในไฟล์ `labai_land_matching.xlsx` มีค่า `-` ใน column เชิงจำนวนถึง **70 จาก 77 แถว** ยังไม่ทราบว่าค่านี้หมายถึง ศูนย์, ไม่มีข้อมูล, หรือสถานะอื่น โปรเจกต์นี้จึงเก็บค่า `-` ไว้เป็น missing value และ **ไม่ใช้ข้อมูลการจับคู่เป็น target** ของโมเดลใดๆ

---

## 4. หน่วยของ area fields ยังไม่ได้รับการยืนยัน

ข้อมูล LABAI มี column ที่เกี่ยวกับพื้นที่ที่ดิน แต่ยังไม่ทราบแน่ชัดว่าหน่วยคืออะไร (ไร่, ตารางเมตร, เฮกแตร์) และช่วงเวลาของการรวบรวมข้อมูล โปรเจกต์นี้จึง **ไม่นำ area fields มารวมกันหรือเปรียบเทียบ** จนกว่าจะยืนยันหน่วยได้

---

## 5. Clustering เป็นการสำรวจ ไม่ใช่การจัดอันดับจังหวัด

การแบ่ง cluster ด้วย KMeans ในโปรเจกต์นี้เป็น **การมองหา pattern เบื้องต้นจาก feature ที่มีอยู่** เพื่อการเรียนรู้เท่านั้น ผลการแบ่ง cluster ไม่ได้บอกว่าจังหวัดใดมีความเสี่ยงสูง-ต่ำ มีประสิทธิภาพดี-แย่ หรือควรได้รับงบประมาณมาก-น้อย

---

## 6. Classification เป็นตัวอย่างการเรียนรู้ ไม่ใช่ระบบทำนายจริง

โปรเจกต์สร้าง label `high_gap` จากค่า quantile 75% ของส่วนต่างผู้ประสงค์ใช้ที่ดินและเจ้าของที่ดิน แล้วนำไปฝึก Logistic Regression เพื่อสาธิต workflow เท่านั้น label นี้ **ไม่ใช่สถานะทางการ** ของจังหวัด และ metric ที่ได้มาจากข้อมูลขนาดเล็ก split เดียว จึงไม่ควรนำไปใช้เป็นเกณฑ์ตัดสินใจ

---

## 7. Regression ถูกข้าม เพราะมีความเสี่ยง data leakage

ตัวแปรที่เป็น target ที่เป็นไปได้มีความสัมพันธ์โดยตรงกับ feature หลายตัว และยังไม่ยืนยันได้ว่า feature ไหนอิสระจาก target เพียงพอ การทำ regression ในสถานการณ์นี้อาจให้ metric ที่ดูดีแต่ไม่สะท้อนความสามารถทำนายจริง

---

## 8. ตัวอย่าง SACIT และ KPI ไม่เปลี่ยนผลของ LABAI

Notebook `07`, `08`, `09` เป็นตัวอย่างแนวทางการทำงานกับชุดข้อมูลอื่น ผลที่ได้จาก SACIT และ KPI **แยกจากการวิเคราะห์หลักของ LABAI** โดยสมบูรณ์ ไม่มีการรวมข้อมูลหรือสรุปข้ามชุดข้อมูล

---

## 9. ผลทั้งหมดมาจากไฟล์ที่ดาวน์โหลด ณ วันที่ 10 กรกฎาคม 2026

ข้อมูลต้นทางอาจมีการอัปเดตจากหน่วยงาน หากดาวน์โหลดใหม่ในภายหลัง ผลการวิเคราะห์อาจแตกต่างออกไป ควรบันทึกวันที่เข้าถึงข้อมูลทุกครั้งที่ดาวน์โหลดซ้ำ
