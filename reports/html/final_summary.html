หน้าแรก รายงานสรุป รายงานข้อมูล interactive รายงาน ML baseline Notebooks

รายงานสรุปโครงการ LABAI

สรุปข้อมูลที่สังเกตได้, baseline ML เพื่อการเรียนรู้, ข้อเสนอเชิงแนวทาง และข้อจำกัดของโครงการ

รายงานใช้ Sarabun จาก `../../Sarabun-Regular.ttf` เมื่อเบราว์เซอร์เข้าถึงไฟล์ได้ หากเข้าถึงไม่ได้ รายงานจะใช้ system font เป็น fallback และยังคงอ่านเนื้อหาได้

Notebook สำหรับอ่านต่อ

Notebookลิงก์เนื้อหา
01 Dataset discoveryเปิด notebook 01ลงทะเบียนชุดข้อมูลและแหล่งข้อมูลทางการ
02 Schema auditเปิด notebook 02ตรวจ row count, column และ missing value
03 Descriptive diagnosticเปิด notebook 03ทำความสะอาดและวิเคราะห์ข้อมูล LABAI
04 Interactive reportเปิด notebook 04สร้างกราฟ interactive ด้วย Plotly
05 Predictive baselineเปิด notebook 05ทดลอง clustering และ classification baseline
06 Prescriptive summaryเปิด notebook 06สรุปข้อเสนอและข้อจำกัดจากผลวิเคราะห์
ตัวอย่างเสริมเปิดโฟลเดอร์ examplesตัวอย่างแนวคิด SACIT และ KPI

ภาพรวมโครงการ

โครงการนี้ใช้ข้อมูลสาธารณะของสถาบันบริหารจัดการธนาคารที่ดินเพื่อสาธิต workflow ตั้งแต่ค้นหาชุดข้อมูล ตรวจ schema ทำความสะอาด วิเคราะห์เชิงพรรณนาและเชิงวินิจฉัย ไปจนถึง baseline ML และข้อเสนอสำหรับการเก็บข้อมูลต่อไป

ผลของข้อมูลและ model ในรายงานนี้มีไว้เพื่อการเรียนรู้และตรวจประเด็นข้อมูลต่อ ไม่ใช้จัดอันดับ ตัดสิน หรือกำหนดการดำเนินงานจริงของจังหวัด

ลำดับการทำงานจากข้อมูลดิบสู่ข้อเสนอ

ขั้นตอน รายละเอียด สถานะ
ค้นหาและคัดเลือกชุดข้อมูล ค้นหาและลงทะเบียน candidate datasets ของ LABAI, SACIT และ KPI พร้อม URL ทางการ เสร็จสิ้น
ดาวน์โหลดและตรวจโครงสร้างข้อมูล ดาวน์โหลดไฟล์ที่เข้าถึงได้และตรวจ schema จริงจากไฟล์ที่อ่านได้ เสร็จสิ้น
ทำความสะอาดและวิเคราะห์ข้อมูล ทำ cleaning, วิเคราะห์เชิงพรรณนาและวินิจฉัย และสร้าง processed CSV เสร็จสิ้น
สร้างรายงานอ่านง่ายและกราฟ interactive สร้างรายงาน interactive ด้วย Plotly สำหรับดูค่ารายจังหวัด เสร็จสิ้น
ทดลอง ML เบื้องต้น ทดลอง clustering และ classification baseline ภายใต้ข้อจำกัดของข้อมูล เสร็จสิ้น
สรุปข้อเสนอและข้อจำกัด สรุปข้อเสนอเชิงแนวทาง, ข้อจำกัด และจัดทำ notebook นี้ เสร็จสิ้น

สรุปการทดลอง ML เบื้องต้น

ตารางนี้อ้างอิงผลการทดลองที่บันทึกไว้แล้วจากการรันโมเดล baseline โดยไม่มีการฝึก model หรือคำนวณ metric ซ้ำในรายงานสรุปนี้

งาน วิธีที่บันทึกไว้ หลักฐานที่บันทึกไว้ ข้อควรตีความ
Clustering KMeans หลัง StandardScaler, เลือก k=3 เพื่อสาธิตการอ่าน profile silhouette ของ k=3 เท่ากับ 0.6751, ขนาดกลุ่ม 15, 60 และ 2 แถว ใช้เพื่อเลือกประเด็นตรวจ metadata และคุณภาพข้อมูลเพิ่ม ไม่ใช่การจัดระดับจังหวัด
Classification Logistic Regression ของ derived label high_gap split เดียว 20 แถว: accuracy 0.850, precision 0.625, recall 1.000, F1 0.769 label ไม่ใช่สถานะทางการ และ metric ไม่ใช่ผลยืนยันการใช้งานจริง
Regression ไม่ดำเนินการ ข้ามเพราะความเหมาะสมของ feature กับ target และความเสี่ยง leakage ยังไม่ยืนยัน ไม่ควรตีความว่าไม่มีความเป็นไปได้ในอนาคต

ข้อเสนอจากผลวิเคราะห์

ข้อเสนอเน้นการยืนยัน metadata และเพิ่มความพร้อมของข้อมูลก่อนขยายการวิเคราะห์หรือประยุกต์ใช้ model

หมวด หลักฐานจากโครงการ ข้อเสนอ
ธรรมาภิบาลข้อมูล นิยามของ count, area และช่วงเวลายังยืนยันไม่ได้ครบจาก metadata ที่ใช้ จัดทำ data dictionary, owner ของข้อมูล, วันที่อ้างอิง และกติกาการปรับปรุงข้อมูลให้ตรวจย้อนกลับได้
คุณภาพข้อมูล ข้อมูลการจับคู่มีค่า placeholder '-' ใน 70 จาก 77 แถว กำหนดรหัส missing หรือสถานะของ '-' อย่างเป็นทางการ แล้วเก็บค่าและเหตุผลแยกจากศูนย์
การวิเคราะห์ ข้อมูลปัจจุบันไม่มี column เวลา และส่วนต่างเป็นผลคำนวณจากจำนวนสอง field เก็บ snapshot ตามเวลาและตรวจหน่วยพื้นที่ก่อนใช้ทำ trend, rate หรือ forecasting
ความพร้อมสำหรับ ML มีเพียง 77 แถวและพบ outlier หลาย field ในขั้นตอนการทดลองโมเดล baseline ใช้ cluster เป็นสัญญาณเลือกประเด็นตรวจเพิ่ม, สร้าง label จากนิยามที่อนุมัติ และเพิ่มตัวแปรอธิบายก่อนทำ model ใหม่
การต่อยอดชุดข้อมูล มีแนวคิด LABAI, SACIT และ KPI ที่ยังไม่ได้ดำเนินการในรายงานนี้ ตรวจ schema และเงื่อนไขการเข้าถึงของ SACIT และ KPI ก่อนต่อยอดเป็นแผนที่ แนวโน้ม หรือข้อมูลเชิงข้อความ

ข้อจำกัดที่ต้องคงไว้ในการตีความ

ข้อจำกัด ผลต่อการใช้งาน
ข้อมูลหลักมี 77 แถวระดับจังหวัด ไม่ควรใช้เปรียบเทียบหรือตัดสินเชิงนโยบาย
ไม่มี column เวลาในไฟล์หลัก ไม่สามารถทำ forecasting ที่มีความหมายจากไฟล์นี้เพียงชุดเดียว
ค่า '-' ในข้อมูลการจับคู่ยังไม่ยืนยันความหมาย ไม่ควรคำนวณอัตราการจับคู่หรือใช้เป็น target
หน่วยและวิธีรวม area fields ยังไม่ยืนยัน ไม่ควรรวมหรือเปรียบเทียบพื้นที่จนกว่าจะยืนยัน metadata
high_gap เป็น label ที่สร้างจาก quantile ไม่ใช่สถานะทางการ ไม่ควรใช้เป็นผลลัพธ์ทางธุรการ
clustering เป็นการสำรวจ pattern เบื้องต้น ไม่ใช่การแบ่งระดับความเสี่ยงหรือศักยภาพ
classification เป็น baseline เพื่อการเรียนรู้จาก split เดียว ไม่ใช่ค่าประสิทธิภาพที่ยืนยันการใช้งานจริง
ข้าม regression เพราะ feature-target suitability และความเสี่ยง leakage ไม่ควรสรุปว่าการพยากรณ์เหมาะสมหรือไม่เหมาะสมอย่างเด็ดขาด

แนวคิดต่อยอดจากชุดข้อมูลอื่น

แนวคิดต่อไปนี้เป็นตัวเลือกจาก `docs/project-idea-gallery.md` ยังไม่ใช่งานที่ดำเนินการในรายงาน LABAI ปัจจุบัน

  • LABAI: เมื่อยืนยันสถานะข้อมูลการจับคู่แล้ว สามารถทบทวน funnel และสำรวจความต้องการกับจำนวนเจ้าของที่ดินตามช่วงเวลาได้
  • SACIT: ชุดข้อมูลการส่งออกและชุมชนหัตถกรรมอาจรองรับ trend, map และ forecasting baseline เมื่อมีช่วงเวลาและ schema จริงที่เพียงพอ
  • KPI: ข้อมูล governance, conflict และ peace อาจใช้ทำ dashboard หรือ map ได้หลังตรวจ resource, เงื่อนไขการเข้าถึง และนิยามตัวชี้วัด
